# 柚子视频

一个可直接部署到 Zeabur 的 Emby 网页版项目。

## 已实现功能

- 默认公共 Emby 线路自动连接，前端不再显示服务器地址、用户名、密码配置 UI
- 媒体库浏览、搜索、排序、正序/倒序
- 最新入库、继续观看、下一集
- 收藏 / 取消收藏
- 标为已看 / 未看
- 剧集详情与按季号、集号自然排序
- 智能播放策略：优先尝试浏览器更容易直接吃下的直链流 + Vidstack 页面内播放
- 若直链失败会自动再试 Emby HLS 转码流；你也可以随时手动切到 mpv 方案
- 播放器容器会根据视频真实宽高比动态渲染，不再强制锁死 16:9 导致大黑边
- mpv 方案仍保留，可填写 `mpv.exe` 路径并复制 PowerShell 播放命令
- 字幕菜单、倍速、清晰度、全屏等播放器界面
- 浏览器直接打开播放直链
- 文本字幕轨自动转换为 VTT 并接入播放器字幕菜单
- 支持浅色 / 深色 / 跟随系统主题

## 技术说明

- 前端：React + TypeScript + Vite
- 播放器：Vidstack
- 部署方式：Zeabur 直接导入仓库
- 不使用本地图片代理，不启动本地后端
- 所有 Emby 请求均由浏览器直接访问服务器

## 重要限制

这是纯网页版本，所以：

1. 默认服务器配置只能做到“前端 UI 隐藏”，不能做到真正保密
2. Emby 服务器必须允许浏览器跨域访问，否则登录后可能无法拉取媒体列表或海报
3. 只有文本字幕（如 SRT / VTT 类）适合前端播放器直接挂载
4. PGS、图形字幕、部分复杂内封 ASS/SSA 往往仍需要 Emby 转码或烧录字幕

## 本地开发

```bash
npm install
npm run dev
```

## 生产构建

```bash
npm run build
npm run start
```

## Zeabur 部署

直接把仓库导入 Zeabur 即可。

推荐配置：

- Install Command: `npm install`
- Build Command: `npm run build`
- Start Command: `npm run start`
- Output: `dist`

仓库已附带 `nixpacks.toml`，一般不用再手填。

## 播放与字幕说明

1. 页面会自动连接默认 Emby 线路
2. 打开媒体详情后点击“检测后播放”
3. 优先进入基于直链流的页面内播放器，右下角设置菜单可切换字幕、倍速、清晰度
4. 如果直链失败，页面会自动再试一次 Emby HLS 转码流，提高很多电影的兼容率
5. 播放器会按视频真实宽高比铺开，避免很多电影在页面内和全屏时出现过大的假黑边
6. 无字幕时也先尝试网页内播放，不会强制立刻回退 mpv
7. 真遇到播放不稳、复杂字幕或浏览器仍然不认的情况，再手动切到 mpv 方案
8. 先在设置页填写 `mpv.exe` 路径，再在详情里点击“复制 mpv 命令”即可本机播放

## CORS 提示

如果登录成功，但媒体列表、海报、继续观看等接口失败，请优先检查：

- Emby 是否允许对应站点跨域
- Cloudflare / 反向代理 是否拦截了浏览器请求
- 目标地址 `https://zhuixin.8622368.xyz:443` 是否稳定可达

我实际探测时，该地址曾返回 Cloudflare 520，所以如果线上偶发打不开，不一定是前端代码问题。
